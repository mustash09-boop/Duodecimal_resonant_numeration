# Public Release Scope v1

## Goal

Publish a small, understandable, reproducible research repository instead of a
raw storage dump.

## Include in GitHub

- core code from `py/`
- supporting scripts from `tools/`
- concise docs from `docs/`
- a few carefully chosen example inputs and outputs
- rendered figures that explain the method
- manifests describing the large offline archives

## Do Not Include in GitHub

- full audio archives
- long-run caches
- giant CSV scans
- intermediate matrix files
- large generated report trees
- private notes that are not part of the scientific story

## Suggested first public demonstration

1. A short single-note instrument comparison
2. A short Bach fragment showing event-count reasoning
3. One ensemble case showing layered instrument-role assignment

## Release posture

Present the project as an emerging research lab:

- interpretable
- reproducible
- exploratory
- technically honest
