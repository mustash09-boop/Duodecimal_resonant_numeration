# How To Run

This is a small inspection-oriented example.

## Step 1

Listen to the source fragment:

- `audio/bach_midi_render_fragment_6p5s_11p5s.wav`

## Step 2

Inspect the reference event table:

- `reference/bach_fragment_midi_events.csv`

## Step 3

Inspect the current pipeline output on the same time window:

- `pipeline/bach_fragment_fused_events.csv`
- `pipeline/bach_fragment_fused_onset_groups.csv`
- `pipeline/bach_fragment_summary.txt`

## What this is meant to show

- reference note events and onset groups
- current fused event structure
- current onset grouping
- the gap between musical reference and current event interpretation

## Important note

This example is intentionally marked as unfinished research.

It should be read as:

- a compact reproducible window into the current state of the system
- not as a final solved polyphonic transcription result
