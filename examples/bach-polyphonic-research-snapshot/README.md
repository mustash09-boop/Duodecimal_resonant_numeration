# Bach Polyphonic Research Snapshot

This example is a compact research snapshot, not a finished public claim of
solved polyphonic transcription.

## Status

`ongoing research`

The purpose of this example is to show:

- a short polyphonic fragment
- the reference MIDI event structure
- the current pipeline event structure on the same time window
- the difference between musical reference and current event interpretation

## Window

- source: `Bach_Invention_1_midi_render.wav`
- fragment: `6.5s -> 11.5s`
- duration: `5.0s`

## Included files

- `audio/bach_midi_render_fragment_6p5s_11p5s.wav`
- `reference/bach_fragment_midi_events.csv`
- `pipeline/bach_fragment_fused_events.csv`
- `pipeline/bach_fragment_fused_onset_groups.csv`
- `pipeline/bach_fragment_summary.txt`

## Why this example matters

This is a compact public case for polyphonic event reasoning:

- the musical material is dense enough to be meaningful
- the files are still small enough to inspect manually
- the example is honest about the current unfinished state of the research
