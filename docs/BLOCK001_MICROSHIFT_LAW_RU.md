# Блок 1. Закон Несхлопывания Микросдвигов

## Статус

Обязательное правило для нового pipeline.

## Формулировка

Микросдвиги нельзя усреднять никогда.

Нельзя:

- рано сводить `note_token` к одному `coarse_note`;
- заменять micro-идентичность усреднённой частотой;
- схлопывать внутреннюю micro-геометрию harmonic chain в один центр;
- считать, что temperированная нота является достаточным представлением реального звука.

Можно:

- хранить `coarse_note` как поздний сопровождающий слой;
- строить coarse-обобщение только как overlay;
- использовать coarse-уровень для позднего сравнения и навигации;
- но не как первичную сущность анализа.

## Почему это обязательно

Чистая temperированная нота является удобной абстракцией, но не реальной звуковой
жизнью.

Реальный звук и гармонический строй существуют только вместе с микросдвигами.

Именно в микросдвигах живут:

- реальные harmonic deviations;
- тонкие отличия между инструментами;
- не-tempered harmonic behavior;
- диезы и бемоли как реальные живые отклонения, а не только табличные знаки;
- переходы между устойчивым тоном и его резонансным окружением.

Если усреднить микросдвиги слишком рано, мы теряем:

- подлинную структуру гармоники;
- различие между близкими нотами и их живыми отклонениями;
- instrument-specific resonance life;
- причинную микрогеометрию события.

## Следствие для нового pipeline

Новый pipeline обязан:

1. хранить `probe_index` как живую сущность;
2. хранить `note_token` micro-уровня как живую сущность;
3. строить event-структуру снизу вверх от micro-life;
4. только потом вводить coarse-слой как позднюю интерпретацию;
5. никогда не подменять micro-структуру единственным усреднённым `weighted_hz`.

## Запрещённые ранние операции

Запрещено на ранних стадиях:

1. группировать событие только по `coarse_note`;
2. считать `weighted_hz` главным представлением ноты;
3. усреднять micro-note path до построения parent/child и lineage;
4. переводить octave-linked microstructure в один общий coarse-узел;
5. отрезать диезно-бемольную жизнь micro-координат ради простоты.

## Разрешённая роль coarse-уровня

`coarse_note` остаётся полезным, но только как:

- поздняя справочная проекция;
- индекс для чтения человеком;
- вспомогательный слой для сравнения с паспортами;
- слой удобной навигации по данным.

Он не является базовой музыкальной сущностью нового анализа.
